```

SPEC 目标: < 1000 ns (1μs) | 状态: ✅ 达标

中位数: 0.0 ns (0.00× SPEC)

**分析**:

- 背靠背写入无累积惩罚
- P99 偶发 42ns 可能来自缓存刷新或调度开销

### 2.2 查询延迟 (Entity::query_state)

```
=== 查询延迟 (空环) ===
中位数: 42.0 ns (0.04× SPEC)
P99: 42.0 ns (0.04× SPEC)

=== 查询延迟 (100 个 delta) ===
中位数: 125.0 ns (0.12× SPEC)

**分析**:

- 100 delta 回放 + 投影 125ns，增量显著

- **退化特征**: 环填充度增加 → 回放循环迭代增加 → 线性退化，但无 O(n²) 突变

## 四、建议

### 4.1 SIMD 性能优化（高优先级）

**问题**: 当前 SIMD 实现在中小型矩阵上反慢

**根本原因**:
1. 矩阵太小（50×10），向量指令开销 > 并行收益
2. `x[col_idx]` 聚集加载（gather）低效
3. 8-lane SIMD 但每行非零元可能 < 8

**优化方向**:
1. **运行时分发**: 根据矩阵特征（NNZ/行、n）动态选择 SIMD/标量
2. **数据布局优化**: 存储转置 CSR 或 ELL 格式提升聚集效率
3. **批量处理**: 累积多行后批量 SIMD 处理摊销开销
4. **更宽向量**: 尝试 16-lane (AVX-512) 提升吞吐

### 4.2 查询性能监控（中优先级）

**建议**:
1. 生产环境监控 `ring_head - ring_tail`（环填充度）
2. 设置告警阈值（如 > 70% 触发快照）
3. 考虑自适应快照策略

### 4.3 级联推理扩展性验证（低优先级）

当前测试最大 500 实体，建议：
1. 增加至 1000/5000 实体测试
2. 验证线性扩展性
3. 测试多跳（10/20 跳）场景
