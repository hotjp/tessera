# Tessera

> 一个确定性的**稀疏图上阈值级联前向模拟器**（sparse-graph threshold-cascade forward simulator）。
> 给定「加权有向图 + 逐节点门控参数 + 初始激励」，在有限跳数内确定性地计算：信号传到哪些节点、多强、第几跳命中、累积时滞多少；以及哪些节点跨越阈值发生「突破」。

Tessera **不绑定任何业务领域**。它是一个纯粹的数值内核：在一张加权有向稀疏图上按跳迭代地做稀疏矩阵–向量乘（SpMV），每跳叠加逐节点的分段门控与幅值剪枝。所有计算是确定性的数值推导——**不是概率模型、不是机器学习、不是黑盒**——每一步可审计、可重放。

## 1. 核心能力（数学定义）

### 1.1 输入

| 符号 | 类型 | 含义 |
|------|------|------|
| `n` | ℕ | 节点数（≤ 65 536） |
| `A` | ℝ^{n×n}，CSR 稀疏 | 耦合矩阵；`A_{uv}=w` 表示一步传播中节点 `u` 接收 `w·(节点 v 的信号)`，即信号沿 **`v → u`** 流动 |
| `bᵢ, τᵢ, dᵢ, λᵢ` | 逐节点 | 门控参数（见 1.3）：脆性标志 / 突破阈值 / 衰减系数 / 每跳时滞 |
| `x⁰ ∈ ℝⁿ` | 向量 | 初始激励（非零分量即「源」） |
| `H ∈ ℕ` | 标量 | 最大跳数（迭代上界，保证终止） |
| `θ ∈ [0,1]` | 标量 | 剪枝阈值（门控置信度 `< θ` 的节点置零、停止传播） |

> **边方向约定**：API `from_edges(n, &[(from=u, to=v, w, lag)])` 把 `(u,v,w)` 写入 `A_{uv}`，语义为信号 `v→u`。故按自然流向书写边时需翻转 `(from,to)`（见 §4 示例）。

### 1.2 传播递推

记 `f⁽ᵏ⁾` 为第 `k` 跳后**继续向前传播**的信号向量（剪枝后），`gᵢ` 为逐节点门控（见 1.3）。令 `f⁽⁻¹⁾ = x⁰`。

**第 0 跳**（处理初始激励）：
```
r⁽⁰⁾ = x⁰
f⁽⁰⁾ᵢ = r⁽⁰⁾ᵢ     if gᵢ(r⁽⁰⁾ᵢ, 0) ≥ θ     else 0
```

**第 k 跳（k = 1..H）**：
```
r⁽ᵏ⁾ = A · f⁽ᵏ⁻¹⁾                         （稀疏 SpMV，线性传播）
f⁽ᵏ⁾ᵢ = r⁽ᵏ⁾ᵢ     if gᵢ(r⁽ᵏ⁾ᵢ, k) ≥ θ     else 0     （剪枝）
```
到 `k = H` 终止。

> **关键性质**：向前传播的幅值是**线性原始值 `r⁽ᵏ⁾ᵢ`，不是门控后的置信度**。门控 `gᵢ` 只用于 (a) 决定剪枝、(b) 产出输出置信度场——它**不进入线性状态演化**。因此「突破饱和为 1.0」不会放大下游幅值（一处已知建模性质，见 `docs/DECISIONS.md` ADR-003 / `.boundary-audit/A6_realism.md`）。

### 1.3 逐节点门控函数 gᵢ

每个节点 `i` 带 `(bᵢ, τᵢ, dᵢ)`，其中 `bᵢ = 1 iff coordinatesᵢ > 0.5`（脆性标志）：

```
gᵢ(r, k) = 1.0              if bᵢ=1 and r ≥ τᵢ      （突破：饱和透传）
          = r · (1/2)ᵏ        if bᵢ=1 and r < τᵢ      （亚阈值：随跳数指数衰减）
          = r · dᵢ            if bᵢ=0                 （线性：每跳乘衰减系数）
```

三类节点行为：
- **脆性·突破**：收到信号 ≥ 阈值 → 置信度饱和为 1.0（继电器/雪崩式元件）；
- **脆性·亚阈值**：收到信号 < 阈值 → 按 `0.5ᵏ` 快速衰减（常被剪枝，切断下游）；
- **非脆性**：按常数 `dᵢ` 线性衰减。

### 1.4 输出

对每个曾被命中（某跳 `gᵢ ≥ θ`）的节点 `i`，产出三张场：

| 字段 | 定义 |
|------|------|
| `hopᵢ` | 首次命中跳数（first-hit depth） |
| `confidenceᵢ` | 历次命中里 `gᵢ` 的**最大值**（峰值） |
| `lagᵢ` | 命中跳数上 `λᵢ` 的**累加和**（含环路 revisit，每命中一次累加一次） |

外加隐含的**突破节点集** `{i : confidenceᵢ = 1.0}`。

## 2. 数学定位（它属于哪一族）

Tessera 是**线性阈值级联模型（linear-threshold cascade）的确定性变体**，与下述同族但相异：

| 模型 | 激活机制 | 随机性 | 与 Tessera 的区别 |
|------|----------|--------|--------------|
| 独立级联 (Kempe '03) | 概率激活边 | 是 | Tessera **完全确定**，无随机边 |
| 线性阈值 (LT) | 加权和 ≥ 阈值 → 激活 | 否 | Tessera 门控只改置信度+剪枝，**不改传播幅值** |
| Bootstrap percolation | 活邻居计数 ≥ 阈值 | 否 | Tessera 用**连续加权和**而非邻居计数 |
| 离散反应-扩散 | 连续浓度衰减 | 否 | Tessera 叠加**阈值饱和**分支 |

**区别性特征**：
1. **确定性**——给定 `(A, params, x⁰, H, θ)` 输出唯一；
2. **逐节点分段门控**，含**饱和突破**分支（继电器元件）；
3. **有界迭代**（≤ H 跳，不追不动点，保证终止）；
4. **幅值剪枝**——每跳按 θ 稀疏化活跃集；
5. **有向、加权、稀疏**（CSR）；
6. **观测与传播解耦**——门控不反馈进线性传播幅值。

## 3. 复杂度与性能

- **每跳**：一次 CSR SpMV `O(nnz)` + 门控/剪枝 `O(n)`，`nnz` = 非零边数。
- **总计**：`O(H · (nnz + n))`。
- SpMV 用 nightly `portable_simd` 8-道向量化（aarch64→NEON/SVE、x86_64→AVX2/AVX-512 自动降级，ADR-001）；标量实现为真值参照。
- 固定预分配内存，运行时无堆增长。

性能目标（`tests/perf_boundary.rs` 守卫）：

| 指标 | 目标 |
|------|------|
| 级联 100 节点 / 5 跳 | < 100 μs |
| 状态查询（无增量回放） | < 500 ns |

## 4. 最小用法

```rust
use tessera::cascade::{cascade, EntityStateView};
use tessera::matrix::CascadeMatrix;

// 5 节点链：信号沿 0→1→2→3→4 流动。
// API 约定 (from=u,to=v,w) 表示 A_{uv}=w 即信号 v→u，故按自然流向需翻转。
fn flow(src: u32, dst: u32, w: f32) -> (u32, u32, f32, u32) { (dst, src, w, 0) }

let matrix = CascadeMatrix::from_edges(5, &[
    flow(0, 1, 0.80),
    flow(1, 2, 0.80),
    flow(2, 3, 0.80),
    flow(3, 4, 0.80),
]);

let mut x0 = vec![0.0f32; 5];
x0[0] = 1.0;                                  // 节点 0 为唯一源，单位激励

let states = vec![EntityStateView {           // 全非脆性，线性衰减 0.8
    coordinates: 0.0, brittle_threshold: 0.0,
    decay_coefficient: 0.8, time_lag_us: 10,
}; 5];

let hit = cascade(&x0, &matrix, &states, 5, 0.10);
// 置信度按 0.8^(k+1) 几何衰减：
//   id0 hop0 conf=0.800, id1 hop1 conf=0.640, id2 hop2 conf=0.512,
//   id3 hop3 conf=0.410, id4 hop4 conf=0.328  （仍 > θ=0.10，命中）
```

**核心 API**：
```rust
pub fn cascade(initial: &[f32], matrix: &CascadeMatrix,
               entity_states: &[EntityStateView],
               max_hops: u32, theta: f32) -> Vec<CascadeResult>;

pub struct EntityStateView { coordinates: f32, brittle_threshold: f32,
                             decay_coefficient: f32, time_lag_us: u32 }
pub struct CascadeResult  { entity_id: u32, confidence: f32, hop: u32, lag_us: u32 }
```

## 5. 它保证什么

- **确定性**：同输入 → 同输出，可重放、可审计（每跳的 `r⁽ᵏ⁾`、剪枝掩码都可检查）。
- **有界终止**：≤ H 跳，无不动点发散风险。
- **白盒**：传播按跳分层可还原（区别于黑盒回归）。
- **微秒级**：稀疏 SpMV + SIMD。

## 6. 它不是什么（防止误用）

| 误解 | 实际 |
|------|------|
| 「置信度 = 概率」 | 否——`confidence` 是传播幅值 ∈ [0,1]，**非后验概率**，无不确定性量化 |
| 「能从数据学出图/参数」 | 否——`A`、权重、阈值、衰减均为**输入**（假设给定），引擎不做推断 |
| 「确定性 = 符合现实」 | 否——确定性只保证可复现；**经验校准是引擎之外的事** |
| 「能给出传播路径」 | 否——输出逐节点聚合场（首跳/峰值/时滞），**非显式路径枚举** |
| 「能反解参数 / 做优化」 | 否——纯前向模拟；无参数拟合、无影响最大化 |
| 「连续时间动力学」 | 否——离散跳；每跳时滞仅用于**累加报告**，不门控传播 |

---

## 7. 作用域与诚实边界

- **所有结构（图、权重、阈值、衰减、激励）都是输入**。引擎不构造图、不赋权、不判定阈值——这些是调用方的责任。引擎只负责：给定这些，确定性地算传播。
- **`CascadeMatrix.time_lag_us`（逐边时滞）当前是存储但未被传播读取的死字段**；只有逐节点 `λᵢ`（`time_lag_us`）参与时滞累加。
- **亚阈值脆性节点的衰减用 `0.5ᵏ`（k 为全局跳数）**——这是一处可改进的建模选择，见 ADR-003。
