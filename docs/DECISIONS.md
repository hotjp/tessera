## ADR-001: SIMD 内核采用 nightly + `std::simd` 可移植 SIMD

**状态**:✅ 已采纳(覆盖 SPEC §6.2 与 附录B 决策树相关条款)

### 背景
- SPEC §6.2 与附录B决策树要求「`target_feature` + x86 intrinsic + 标量 fallback」,并明确警告「不要直接写 `std::simd`(不稳定)」。
- 原任务 `task_008` 设计为 x86_64 AVX-512/AVX2 intrinsics + `is_x86_feature_detected!!` 运行时分发。
- 用户要求系统具备**跨平台**能力(见 ADR-002 的 4 个目标平台)。在 4 个架构上维护两套手写 intrinsic(x86_64 + aarch64)成本与出错率高于接受一份可移植代码。

### 决策
- SIMD 内核改用 **nightly Rust + `#![feature(portable_simd)]` + `std::simd`**(如 `std::simd::Simd<f32, 16>` + gather)实现 SpMV。
- **一份代码**自动降级到各平台 SIMD:x86_64 → AVX-512/AVX2,aarch64 → NEON/SVE。
- **不再使用**任何 `std::arch::x86_64::*` 专属 intrinsic 与 `is_x86_feature_detected!`。
- `task_007` 的**标量 SpMV 保留**,作为正确性 oracle;测试断言 SIMD 路径结果 == 标量(误差 < 1e-5)。

### 理由
4 个目标平台用一份可移植代码,优于手写并维护 x86_64 + aarch64 两套 intrinsic 内核;`portable_simd` 成熟度已可承担本工作负载。

### 后果 / 代价
- 工具链必须钉 nightly(`rust-toolchain.toml`:带日期的 nightly + `rust-src` 组件),由 `task_001` 产出。
- 4 平台 CI 均需安装该固定 nightly(`task_012`)。
- `portable_simd` 仍为 unstable:钉版本后风险可控,但**升级 nightly 前必须跑全量回归**。
- **性能不受影响**:目标负载(100 实体 / 5 跳稀疏 SpMV)标量即可 < 100μS,SIMD 非承重墙——跨平台不以性能为代价。

### 覆盖范围
`task_008`(实现)、`task_001`(工具链)、`task_012`(CI)。

## ADR-002: 目标平台矩阵

**状态**:✅ 已采纳

### 决策
正式支持的构建 / 测试平台:
1. **aarch64 macOS**(Apple Silicon,开发机)
2. **x86_64 Linux**(服务器部署)
3. **aarch64 Linux**(ARM 服务器 / AWS Graviton)
4. **Windows**(x86_64)

### 后果
- CI 矩阵须覆盖上述 4 平台(`task_012`)。
- 所有文件路径(TLS 证书 / 日志 / 快照)**必须用 `std::path::PathBuf`**,禁止硬编码 POSIX 路径如 `/var/lib/...`(影响 `task_010` 及快照层)。
- 传输层 **rustls + tokio** 在 4 平台原生可用(Windows 异步 IO 走 IOCP),无需额外适配。
- 线协议 `to_be_bytes`(大端)与快照 `to_le_bytes`(小端)在上述架构上一致,无需调整。

## ADR-003: 脆性语义现实性修正（突破放大 + 子阈值传导 + 韧性解耦）

**状态**:📋 提案(未采纳;待落地进 `src/cascade.rs` 后激活。实验证据见 `.boundary-audit/A6_realism.md`,设计见 `docs/EXPERIMENT_brittleness_realism.md`)

### 背景
**领域定位**:本引擎是**领域无关的级联推理引擎**——「实体 + 加权影响边 + 脆性门控」是一套抽象动力学,适用于任何「冲击沿网络传导、节点带临界点」的场景。金融传染与舆情研判是**同构**的两个实例,不是垂直专用:

| 抽象概念 | 金融场景 | 舆情场景 |
|---------|---------|---------|
| 实体 Entity | 基金 / 主权基金 / 家办 | 大V / 媒体 / 社区 / 平台 |
| 影响边 Edge | 持仓 / 信用 / 因子敞口 | 关注 / 转发 / 引用 / 立场耦合 |
| 冲击注入 | 赎回 / 爆仓事件 | 爆料 / 负面帖 / 话题 |
| 脆性 = 引爆敏感度 | 高杠杆 + 集中仓位 | 立场极化 + 易被点燃 |
| 突破放大 | 被迫平仓砸盘 | KOL/社区被点燃后主动带节奏 |
| 韧性吸收 | 资本充足主体接盘 | 权威辟谣 / 官方定调 / 冷静节点 |
| 系统性风险 | 金融危机 | 舆情危机 / 群体极化 / 品牌风暴 |

**跨域实验已验证**:S2(金融赎回潮)与 S3(舆情爆料级联)采用**同构拓扑**、同套 V0-V4 变体,产出同构动力学(突破放大比 金融 2.15× / 舆情 1.42×,见 A6 §跨域对照)。引擎核心无需为任一领域定制。

`src/cascade.rs` 现行脆性模型存在 3 处与规律相悖的 P1 缺陷(**适用于一切领域**,详见 A6 报告):
  1. **突破不放大下游**:`apply_hop` 中 `propagated[i]=raw`(`cascade.rs:84`)——脆性突破只把节点自身置信度置 1.0,**出向幅度不变**,系统性风险被低估。
  2. **sub-threshold 黑洞**:`raw × 0.5^hop`(`cascade.rs:50`)常低于 θ 被剪枝 → **切断下游**,脆性节点被当成完美阻断器。
  3. **脆性/吸收语义耦合反转**:同一段 `coordinates>0.5` 分支既做阈值门控又(经剪枝)充当吸收,使「脆性·高阈值」误充安全阀。
- SPEC 自己已预警:`SIGMA4_SPEC_v1_1.md` 附录 A(line 1025)「脆性实体统一衰减 → LTCM 式崩溃被低估」。
- 外置实验(`examples/brittleness_experiment.rs`,5 变体 × 2 场景;V0 精确复现 demo)证实:V0 下突破/不突破的下游 conf 与 energy **完全相等**;引入 V1+V3+V2(V4)后,方案 A terminal#7 由 0.286 升至 0.428,S2 循环拓扑下 energy 达 V0 的 **2.15×**(复现传染螺旋)。

### 决策(提案)
1. **突破放大**:脆性突破时出向 `propagated = raw × amp`;新增字段 `EntityStateView.breach_amplification: f32`(**默认 1.0 = 现状兼容**)。
   - 金融:杠杆节点爆仓 → 被迫平仓向市场砸出更大卖压。
   - 舆情:极化社区/KOL 被点燃 → 主动转发、带节奏,辐射量远超被动接收。
2. **子阈值传导**:脆性未突破时置信度按 `raw × sub_coef` 计(替代 `raw × 0.5^hop`);新增字段 `EntityStateView.sub_threshold_coef: f32`(**默认 0.0 = 现状快速衰减,向后兼容**)。
   - 金融:未爆仓的高杠杆机构仍在抽流动性、收紧信用。
   - 舆情:立场未达引爆点的节点仍在散播情绪、影响观望者。
3. **韧性解耦**:新增字段 `EntityStateView.resilience: f32`(`[0,1]`,**默认 0.0**),独立衰减出向 `× (1−ρ)`;**吸收角色与脆性正交**——吸收冲击的应是韧性主体,不是脆性主体。
   - 金融:资本充足的主权基金/大型接盘机构。
   - 舆情:权威辟谣节点、官方定调、可信度高的冷静媒体。
4. **推荐参数**(经验值,未校准):`amp ∈ [1.3, 2.0]`(实验用 1.5)、`sub_coef ∈ [0.2, 0.4]`(实验用 0.3)、`resilience` 应配置在**高中介度节点**上(A6 发现 6:「吸收器放哪 > ρ 多大」——金融放大型接盘机构、舆情放高公信力辟谣节点)。
5. **不改** `propagated[i]=raw` 为 `propagated[i]=conf` 的备选——会破坏剪枝语义;放大与剪枝必须解耦(放大改出向,剪枝看置信度)。
6. **字段命名**:新增三字段名(`breach_amplification` / `sub_threshold_coef` / `resilience`)均为领域无关术语;现有 `brittle` 取材料学通用义(脆性=引爆敏感度),不绑定金融。各领域在**数据层**(实体名、边语义、参数取值)各自解释,引擎 API 不变。
7. 可选:运行时死字段 `CascadeMatrix.time_lag_us` 可复活为逐边放大载波,但本期建议先落在 `EntityStateView`(逐实体),逐边放大留作后续。

### 理由
- 实验 H1/H3 **强支持**:V1 使下游 conf +49%、V3 恢复 sub-threshold 传导(reach 6/8→8/8);S2 循环拓扑 V1 energy 2.15× 复现传染螺旋——正是 SPEC 附录 A 所忧。
- **跨域同构证据**:S3 舆情场景(与 S2 同构)下,V1 突破放大比 1.42×、V2 辟谣吸收使 reach 11→10 / energy −13%——同一套修正对舆情研判同样生效,证明非垂直专用。
- H2 **方向支持**(幅度随拓扑位置变化):暴露「吸收器放哪 > ρ 多大」规律,本身是有用设计结论(金融/舆情均适用)。
- **向后兼容**:三个新字段默认值(1.0 / 0.0 / 0.0)使旧调用方行为不变,可平滑引入、灰度验证。

### 后果 / 代价
- `EntityStateView` 由 4 字段增至 7 字段(仍 `#[repr(C)]`、`Copy`),内存影响可忽略。
- 现有 67 个单元测试中 `cascade::*` 需复核(默认值应保持原结果);新增 V1/V3/V2 回归用例。
- SPEC §6.3 伪代码需同步:突破返回 `1.0` vs 伪代码的 `impact` 已有分歧,借此一并对齐。
- **参数未校准**:真实校准需 CSV 结构化边表(当前无),列为后续。

### 覆盖范围
**未来任务**(落地 `src/cascade.rs`):新增 3 字段、改写 `apply_hop`/`confidence_for`、更新 SPEC §6.3、补回归测试。**本次仅产出提案,不改代码。**

**多领域接入**:引擎核心(`CascadeMatrix` / `cascade` / `EntityStateView`)领域无关;领域差异只在**数据层**——实体命名、边语义、参数取值由各领域的 loader/配置提供(金融用 183 资本主体 CSV,舆情另备 KOL/社区图)。引擎 API 与本 ADR 的修正对所有领域统一生效。
