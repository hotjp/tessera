# 实验设计：脆性语义的现实性检验

> **状态**：实验已执行（2026-07-13）。数据见 [`.boundary-audit/A6_realism.md`](../.boundary-audit/A6_realism.md)；设计规范草案见 [`DECISIONS.md`](DECISIONS.md) `ADR-003`。
> **范围**：**外置实验 harness，不改生产引擎**。引擎落地为后续独立任务。

## 1. 问题陈述

`examples/systemic_risk_demo.rs` 的虚构场景（赎回潮级联）暴露了 `src/cascade.rs` 脆性模型与社会规律的两处实质性背离，且均有代码根因：

| # | 背离 | 代码根因 | 社会规律 |
|---|------|----------|----------|
| **B1** | 突破不放大下游 | `apply_hop` 中 `propagated[i] = raw`（`cascade.rs:84`）——脆性突破只把节点自身置信度置 1.0，**出向信号幅度不变** | Archegos 式崩溃相反：突破的杠杆节点被迫平仓，向市场砸出**更大**卖压（二阶放大） |
| **B2** | 「脆性·高阈值」反成吸收器 | sub-threshold 走 `raw × 0.5^hop`（`cascade.rs:50`）→ 常被剪枝 → 切断下游 | 吸收冲击的是**资本充足、韧性强**的主体，不是脆性主体——语义反转 |

**规范立足点**：`SIGMA4_SPEC_v1_1.md` 附录 A（line 1025）已预警「脆性实体统一衰减 → LTCM 式崩溃被低估」，附录 B 决策树（line 1069）强制要求处理脆性。这是 SPEC 作者自己埋的已知雷。

## 2. 假设

| 假设 | 内容 | 判据 |
|------|------|------|
| **H1** | 当前引擎（V0）突破**不放大**下游；引入突破放大（V1）后下游置信度上升 | V0：方案A(#6 突破) 与 方案B(#6 非脆) 的 terminal#7 与 energy **相等**；V1：方案A terminal#7 > V0 |
| **H2** | 韧性吸收器（V2）**降低**下游 stress | S2 中 ρ=0.7 的吸收器使 energy < ρ=0.0 |
| **H3** | 子阈值传导（V3）**消除**脆性节点的「干净黑洞」 | 方案C(#6 难破)：V0 terminal#7 未命中 → V3 命中 |
| **H4** | 组合修正（V4）在含循环的扩展拓扑上产出符合直觉的传染图谱 | S2：吸收器降低 energy、脆性突破放大 energy |
| **H5** | 引擎**领域无关**——同构拓扑在金融与舆情两域产出同构动力学（非垂直专用） | S3（舆情）的 V1 突破放大比、V2 吸收效果与 S2（金融）同向 |

## 3. 变量

### 3.1 自变量：5 个语义变体

vendor `src/cascade.rs` 的 `confidence_for` / `apply_hop` / `cascade` 到 harness（`examples/brittleness_experiment.rs`），在副本上靠三个旋钮分叉：

| 变体 | `amp`（突破放大） | `sub_coef`（子阈值传导） | `use_resilience`（韧性吸收） |
|------|------------------|------------------------|----------------------------|
| **V0 现状（对照）** | 1.0（关） | 0.0（关，走 `0.5^hop`） | 否 |
| **V1 突破放大** | 1.5 | 0.0 | 否 |
| **V2 韧性吸收** | 1.0 | 0.0 | 是（读 `resilience ρ`） |
| **V3 子阈值传导** | 1.0 | 0.3 | 否 |
| **V4 组合** | 1.5 | 0.3 | 是 |

变体化的出向幅度公式：
```
mag = raw
if 脆性突破 and amp>1:  mag *= amp            # V1
if use_resilience:      mag *= (1 − ρ)        # V2，与 V1 复合
propagated[i] = mag
```

### 3.2 因变量：3 个场景（含跨域）

> **领域定位**：引擎核心领域无关。S2（金融）与 S3（舆情）采用**同构拓扑**验证同一套动力学对两个领域统一生效——避免把引擎做成垂直专用工具。

- **S1 基线（8 实体，金融）**：复用 `systemic_risk_demo.rs` 拓扑；#6 取三种画像（脆性·易破 / 非脆性 / 脆性·难破）。**校验锚点**：V0 必须复现 demo 的 A/B/C 真实数字。
- **S2 扩展（14 实体，金融）**：① 主权基金吸收器（ρ=0.7）② 双脆性杠杆节点（家办 A/B）③ **回边循环** `5↔6`（静态矩阵下近似反馈）④ 第二冲击源。验证「韧的吸收、脆的放大」。
- **S3 舆情研判（13 实体，跨域）**：与 S2 **同构**、领域换为舆情——爆料源 / 大V / 极化社区（脆性·低阈值=易点燃）/ 官方辟谣节点（吸收器 ρ=0.7）/ 回声室循环 / 第二爆料源 / 大众终端。证明 V0-V4 对舆情研判同样生效。

## 4. 指标

| 指标 | 定义 | 意义 |
|------|------|------|
| `reach` | 命中实体数 / 总数 | 传染广度 |
| `terminal_hit` / `terminal_conf` | 终端养老金是否被波及 + 置信度 | 社会终端冲击 |
| `brittle_conf` | 脆性节点置信度 | 是否被标记为系统级关键节点 |
| `energy` | Σ_各跳 Σ_i 出向信号幅度 | **系统性应力积分（新指标）**——V0 下突破/不突破应相等（诊断 B1） |
| `amp_ratio` | energy(突破开) / energy(突破关) | V0 应 ≈1.0；V1/V4 应 >1 |

## 5. 方法

1. **vendor 策略**：把 `confidence_for`（`cascade.rs:42-53`）+ `apply_hop`（`:59-89`）+ `cascade`（`:100-155`）共约 60 行复制进 `examples/brittleness_experiment.rs`，在副本上分叉变体。生产引擎零改动，**217 测试不受影响**。
2. 只调用 public API：`spmv_csr`（`matrix.rs:95`）、`CascadeMatrix::from_edges`（`:29`）、`EntityStateView`、`CascadeResult`。
3. 韧性吸收经并行 `Vec<f32>` 传入（不复用 `CascadeMatrix.time_lag_us` 死字段，保持语义清晰）。
4. 固定 θ=0.20、max_hops=5（与 demo 一致）。

| 假设 | 预期 | 实际（见 A6） |
|------|------|---------------|
| H1 | V0 A/B terminal 相等；V1 A 上升 | ✅ V0 均为 0.286 / energy 均为 5.096；V1 A 升至 0.428 |

## 7. 局限

- **数值未校准**：AMP=1.5、SUB=0.3、ρ=0.7 为经验取值，非数据拟合。CSV 无结构化边，真实校准为后续。
- **静态矩阵**：回边循环仅近似反馈；真正的价格中介反馈环需架构级改动。
- **H2 弱**：吸收器在低吞吐路径上效果有限——说明「吸收器放哪」比「ρ 多大」更关键（这本身是有用结论）。
